Automation

The day of manually testing, building and publishing your applications are long gone (if they ever existed). Application development is getting complex, and each of its phases, from conception to delivery, is full of subparts. In the JavaScript world, several tools must be played interwined, to test, assembly, compile, minify, lint and deliver your code. That is why you need automation.

Brocolli

A build tool on Node.js that process your source files as a dependency tree, with support for partial re-build on file changes.

Links

  • https://github.com/broccolijs

Grunt

JavaScript build tool, on top of Node.js, with great community support and several plugins to process your source files into assets

Links

  • http://gruntjs.com/

Gulp

Build tool in JavaScript that lets you process your source files as streams, progmatically, to build your assets

Links

  • http://gulpjs.com/

Make

Build tool to process source files and build assets. It is language agnostic and let you transform any type of file.

Links

  • www.gnu.org/software/make/
